# Implementation Workflow

This directory contains the structured implementation workflow for IMC Pelita Logistik KMS.

## Folder structure

```
docs/implementation/
├── README.md              ← You are here
├── workflow.md            ← Why we use 4 separate sessions
└── slices/
    └── <slice-name>/
        ├── spec.md        ← What to build (you write this first)
        ├── plan.md        ← Generated by /plan
        ├── review.md      ← Generated by /review
        └── completed.md   ← Generated by /test-commit
```

## How to use

### 1. Spec (`/spec <slice-name>`)

Run `/spec <slice-name>` to start. Claude will interview you about what you want to build, ask clarifying questions, and produce `spec.md` with user stories, acceptance criteria, and scope boundaries.

### 2. Plan (`/plan <slice-name>`)

Run in a fresh Claude Code session. Reads the spec and produces `plan.md` with files to create/modify, schema changes, API endpoints, components, and a manual test walkthrough.

Review the plan. Edit if needed. Approve before proceeding.

### 3. Implement (`/implement <slice-name>`)

Run `/clear` first, then run in a fresh session. Follows the approved plan exactly. Writes code to disk without committing. Runs `tsc` and `lint` before declaring done.

### 4. Review (`/review <slice-name>`)

Run `/clear` first, then run in a fresh session. Acts as a hostile code reviewer. Reads the diff against spec/plan (not the implementer's reasoning). Produces `review.md` with findings categorized as 🔴 Blocker, 🟡 Important, or ⚪ Nit.

### 5. Test & Commit (`/test-commit <slice-name>`)

Run `/clear` first, then run in a fresh session. Fixes blockers, walks through manual testing with you, writes `completed.md`, and commits.

## Rules

- **One slice = one commit batch.** Don't mix slices.
- **Fresh session between phases.** Use `/clear` to avoid context exhaustion and confirmation bias.
- **No code in planning.** No implementation in review. Each phase has a single responsibility.
- **Stop at slice boundary.** Don't start the next slice in the same session.
