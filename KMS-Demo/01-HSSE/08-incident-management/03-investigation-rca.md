# Investigation & Root Cause Analysis

## Purpose

This document provides procedures for investigating incidents and conducting Root Cause Analysis (RCA) to identify underlying causes and prevent recurrence.

---

## Scope

Applies to: **All significant incidents requiring investigation**

---

## When Full Investigation Required

| Incident Type | Investigation |
|---------------|---------------|
| Fatality | Full investigation + external (if required) |
| Permanent disability | Full investigation |
| High Potential (HiPo) | Full investigation |
| LTI | Standard investigation |
| Repeat incidents | Standard investigation |
| Major property damage | Standard investigation |
| Tier 2/3 environmental | Full investigation |

## 1. Secure the Scene

| Action | Purpose |
|--------|---------|
| Ensure safety | No further injury |
| Restrict access | Preserve evidence |
| Photograph/video | Visual record |
| Collect physical evidence | Parts, equipment |
| Record positions | Where things/people were |

## 2. Gather Evidence

### Witness Interviews

| Guideline | Details |
|-----------|---------|
| Interview separately | Prevent influencing each other |
| As soon as possible | Memory fades quickly |
| Open questions | "Tell me what happened" |
| No blame | Focus on facts, not fault |
| Written statement | Signed and dated |

### Documentary Evidence

| Evidence | Source |
|----------|--------|
| Work permits | Permit to work records |
| Risk assessments | JSA, HAZID |
| Training records | Crew training files |
| Maintenance records | PMS, work orders |
| Log books | Deck log, engine log |
| Watch schedules | Work/rest records |

### Physical Evidence

| Evidence | Handling |
|----------|----------|
| Failed equipment | Retain for analysis |
| PPE | Preserve if relevant |
| Samples | Oil, chemicals |
| Photos | Date/time stamped |

## 3. Root Cause Analysis (RCA)

### The "5 Whys" Technique

Keep asking "Why?" until you reach the root cause:

**Example:**
1. **Why** did the crewman get injured?
   - He slipped on the deck
2. **Why** did he slip?
   - There was oil on the deck
3. **Why** was there oil on the deck?
   - A hydraulic line leaked
4. **Why** did the hydraulic line leak?
   - It wasn't replaced during maintenance
5. **Why** wasn't it replaced?
   - The maintenance schedule wasn't followed

**Root Cause:** Failure in maintenance management system

### Cause Categories

When analyzing, consider:

| Category | Examples |
|----------|----------|
| **Immediate/Direct** | What directly caused harm |
| **Underlying** | Conditions that allowed it |
| **Root Cause** | System/management failures |

### Common Root Cause Categories

| Category | Examples |
|----------|----------|
| **Procedures** | Missing, unclear, not followed |
| **Training** | Inadequate, not refreshed |
| **Supervision** | Insufficient, not present |
| **Equipment** | Failed, poorly maintained |
| **Communication** | Not clear, not received |
| **Work environment** | Poor lighting, housekeeping |
| **Management** | Resources, culture, systems |

## 4. Corrective Actions

### SMART Actions

| Criterion | Meaning |
|-----------|---------|
| **S**pecific | Clear what needs to be done |
| **M**easurable | Can verify completion |
| **A**ssignable | Named responsible person |
| **R**ealistic | Can be achieved |
| **T**ime-bound | Deadline set |

### Hierarchy of Controls

Prioritize corrective actions in this order:

1. **Eliminate** – Remove the hazard entirely
2. **Substitute** – Replace with less hazardous
3. **Engineering** – Physical barriers/controls
4. **Administrative** – Procedures, training
5. **PPE** – Personal protection (last resort)

### Action Tracking

| Element | Details |
|---------|---------|
| Action description | What specifically |
| Responsible person | Named individual |
| Due date | Realistic deadline |
| Status | Open/In Progress/Closed |
| Verification | How confirmed complete |
